Detailed Description

It defines all the different parameters used in the MPD algorithm.

Field Documentation

double delta_t

Only used if ls_mode=0: in advect command line (software must have been previously installed), it represents the final time at which the advection equation is solved: it must be positive.

double delta_x

Discretization step in the first direction: we must have delta_x==(x_max-x_min)/(double)(n_x-1).

double delta_y

Discretization step in the second direction: we must have delta_y==(y_max-y_min)/(double)(n_y-1).

double delta_z

Discretization step in the third direction: we must have delta_z==(z_max-z_min)/(double)(n_z-1).

double hausd_iso

Only used if ls_mode=0: tolerance ruling the error required for the Hausdorff distance between the adapted surface and its ideal representationi during the computation of the default isotropic adaptation metric by mmg3d software, which must have been previously installed; it must be positive.

double hausd_ls

Only used if ls_mode=0: tolerance ruling the error required for the Hausdorff distance between the adapted surface and its ideal representationi during the computation of both level-set function and molecular orbitals adaptation metric by the modified version of mmg3d software, which must have been previously installed; it must be positive.

double hausd_met

Only used if ls_mode=0: tolerance ruling the error required for the Hausdorff distance between the adapted surface and its ideal representationi during the computation of the molecular orbitals adaptation metric by mmg3d software, which must have been previously installed; it must be positive.

double hgrad_iso

Only used if ls_mode=0: tolerance allowed for the ratio between any two edge lengths sharing one point together during the computation of the default isotropic adaptation metric by mmg3d software, which must have been previously installed; it must be (strictly) greater than one.

double hgrad_ls

Only used if ls_mode=0: tolerance allowed for the ratio between any two edge lengths sharing one point together during the computation of both level-set function and molecular orbitals adaptation metric by the modified version of mmg3d software, which must have been previously installed; it must be (strictly) greater than one.

double hgrad_met

Only used if ls_mode=0: tolerance allowed for the ratio between any two edge lengths sharing one point together during the computation of the molecular orbitals adaptation metric by mmg3d software, which must have been previously installed; it must be (strictly) greater than one.

double hmax_iso

Only used if ls_mode=0: maximum length allowed for edges in the computation of the default isotropic adaptation metric by mmg3d software, which must have been previously installed; it must be (strictly) greater than hmin_iso.

double hmax_ls

Only used if ls_mode=0: maximum length allowed for edges in the computation of both level-set function and molecular orbitals adaptation metric by the modified version of mmg3d software, which must have been previously installed; it must be (strictly) greater than hmin_ls.

double hmax_met

Only used if ls_mode=0: maximum length allowed for edges in the computation of the molecular orbitals adaptation metric by mmg3d software, which must have been previously installed; it must be (strictly) greater than hmin_met.

double hmin_iso

Only used if ls_mode=0: minimum length allowed for edges in the computation of the default isotropic adaptation metric by mmg3d software, which must have been previously installed; it must be positive and not (strictly) greater than hmax_iso.

double hmin_ls

Only used if ls_mode=0: minimum length allowed for edges in the computation of both level-set function and molecular orbitals adaptation metric by the modified version of mmg3d software, which must have been previously installed; it must be positive and not (strictly) greater than hmax_ls.

double hmin_met

Only used if ls_mode=0: minimum length allowed for edges in the computation of the molecular orbitals adaptation metric by mmg3d software, which must have been previously installed; it must be positive and not (strictly) greater than hmax_met.

int iter_max

Maximum number of iterations allowed in the optimization loop: it must not be negative (zero means no loop is performed).

double iter_toldp

Tolerance on the shape derivative (if ls_mode=(+/-)1, between the probability differences of two successive iterations): it it must not be negative (zero means it is not taken into account).

double iter_tolp

Tolerance between the probabilities of two successive iterations: it must not be negative (zero means it is not taken into account).

int ls_mode

If set to (+/-) one, then the cubic algorithm is performed with level-set/random perturbations; otherwise, it must be set to zero and the tetrahedral algorithm is performed.

double ls_r

Length/Radius of the cube/sphere associated with the initial domain: it must be positive.

int ls_type

If set to zero, then the initial domain is a cube of center (ls_x,ls_y,ls_z) and size ls_r; otherwise, it must be set to one and the initial domain is a sphere (same center and radius ls_r).

double ls_x

First coordinate of the center of the initial cube/sphere.

double ls_y

Second coordinate of the center of the initial cube/sphere.

double ls_z

Third coordinate of the center of the initial cube/sphere.

double met_err

Only used if ls_mode=0: tolerance ruling the error required for the computation of the adaptation metric; it must be positive.

double met_max

Only used if ls_mode=0: maximum length allowed for edges in the computation of the adaptation metric: it must be (strictly) greater than met_min.

double met_min

Only used if ls_mode=0: minimum length allowed for edges in the computation of the adaptation metric; it must be positive and not (strictly) greater than met_max.

int n_cpu

Number of cpu/threads required for the parallelization by openmp in the MPD program (also used by mshdist software, which must have been previously installed): it must be positive.

int n_iter

Only used if ls_mode=0: in mshdist command line (software must have been previously installed), it represents the maximum number of iterations allowed in the iterative renormalisation of the signed distance function; it must not be negative (zero means that no limit is fixed).

int n_x

Number of points in the first direction: it must be (strictly) greater than two (it allows at least one normal/tangent vector on the cube faces).

int n_y

Number of points in the second direction: it must be (strictly) greater than two (it allows at least one normal/tangent vector on the cube faces).

int n_z

Number of points in the third direction: it must be (strictly) greater than two (it allows at least one normal/tangent vector on the cube faces).

char* name_chem

Used to save the name of the *.wfn/ *.txt file containing the informations needed for filling ChemicalSystem structure.

char* name_elas

Only used if ls_mode=0: used to save the name of the *.elas file containing the informations needed for the commandline of elastic software (warning: it must have been previously installed).

char* name_info

Used to save the name of the *.info file containing the informations needed for filling the Parameters structure.

int name_length

Length necessary to store the different file names: it must be (strictly) greater than six (to be able to store at least a name that contains something more than *.info format).

char* name_mesh

Used to save the name of the *.mesh/ *.cube file containing the informations needed for filling the Mesh structure.

int no_cfl

Only used if ls_mode=0: in advect command line (software must have been previously installed), if it is set to zero, then a cfl condition is computed, restricting the final time in the advection equation; otherwise, it must be set to one and no restriction is performed (but instability can appear).

int nu_electrons

The number of electrons to look for: it must be positive (zero returns an error) and less than the total number of orbitals.

int orb_spin

If set to one, then it duplicates the molecular orbitals whose occupation number equals two in the *.wfn file; otherwise, it must be set to zero.

char* path_advect

Only used if ls_mode=0: path specification for advect software which must have been previously installed.

char* path_elastic

Only used if ls_mode=0: path specification for elastic software which must have been previously installed.

int path_length

Length necessary to store the different path names: it must be (strictly) greater than one (to store more than the terminating nul character '\0').

char* path_medit

Path specification for medit software which must have been previously installed.

char* path_mmg3d

Only used if ls_mode=0: path specification for mmg3d software which must have been previously installed.

char* path_mshdist

Only used if ls_mode=0: path specification for mshdist software which must have been previously installed.

int save_data

Frequency at which data are saved (probability, population, probability difference, shape derivative) in the loop: it must not be negative (zero means data are never saved).

int save_mesh

Frequency at which the mesh is saved in the optimization loop: it cannot be negative (zero means the mesh is never saved).

int save_print

Frequency at which the mesh is plotted with medit (warning: software must have been previously installed and every time medit is loaded, algorithm will stop until exit is performed): it must not be negative (zero means no plot of the mesh).

int save_type

Only used if ls_mode=(+/-)1: if set to one, then the mesh is saved using the *.mesh format; if set to zero, then the mesh is saved using the *.cube format; otherwise, it must be set to two, and in this case, mesh is saved in both *.cube and *.mesh format.

int save_where

Only used if ls_mode=0: integer referring to which part of the loop save_print is vizualizing: 1=metric, 2=level-set function, 3 or 4=shape derivative (before/after relabelling), 5=extension of shape derivative, 6=advection of level-set function, and 7=advected domain; other values are forbidden.

int trick_matrix

Only used if ls_mode=(+/-)1: if set to zero, then the overlap matrix is completely recomputed at each iteration; otherwise, it must be set to one and a trick is performed at each iteration (except for the first one) on the hexahedron's labels to compute only the difference between the old overlap-matrix coefficients and the new ones.

int verbose

If set to one, then it prints every execution details in the standard output stream; otherwise, it must be set to zero and very little is shown during the execution of the algorithm.

double x_max

Maximal coordinate in the first direction: it must be (strictly) greater than x_min.

double x_min

Minimal coordinate in the first direction: it must not be (strictly) greater than x_max.

double y_max

Maximal coordinate in the second direction: it must be (strictly) greater than y_min.

double y_min

Minimal coordinate in the second direction: it must not be (strictly) greater than y_max.

double z_max

Maximal coordinate in the third direction: it must be (strictly) greater than y_min.

double z_min

Minimal coordinate in the third direction: it must not be (strictly) greater than z_max.
